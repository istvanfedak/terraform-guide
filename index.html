setup

creating a new terraform directory

create a new directory, open it and create a provider.tf file
enter one of the following cloud poviders into the file and save
provider "aws" {
    region = "us-east-1"
}

provider "azurerm" {
    features {}
}

provider "google" {
    project = "my-project-id"
    region = "us-central1"
}

View all providers

initialize a new directory

create a new directory, open it and perform a
terraform init
to initialize the terraform project

blocks

Blocks are used to define the configuration of some kind of object,
like a resource. Blocks have a block type, can have
zero or more labels, and have a body that contains any
number of arguments and nested blocks

<block type> "<block label>" "<block label>" {
    <argument>
}

The ordering of blocks and the files they are organized into are
generally not significant
